10.18 传统艺能（概率巧题、计数、计算贡献）

因为随机分成两个集合，再随机取过于复杂，考虑可不可以化为一个简单的模型：两个人在序列中随机取数

进一步地，将取出的数记为一个序列，就变成了：一个序列中每两个数中前面一个数比后面一个数大的期望个数

考虑计算贡献，记这样的数对数为cnt，因为在序列中出现的位置不确定，所以将每个位置出现的同一个数对记为不同，所以就有了cnt∗n/2cnt * n / 2cnt∗n/2中本质不同的数对

考虑每种出现的次数，为其他位置随便放(n−2)!(n-2)!(n−2)!

将每种的贡献（为1）乘上出现次数加起来，再比上总情况数n!n!n!即可


const int N (1000005);
const int mod (998244353);

int a[N], b[N], tmp[N], sum[N];
LL cnt, jc1 = 1, jc2 = 1;

inline LL power(LL a, LL b)
{
    LL res = 1;
    for(; b; b >>= 1, a = 1LL * a * a % mod)
        if(b & 1) res = 1LL * res * a % mod;
    return res;
}

signed main()
{
    int n (read());
    for(int i (1); i <= n; ++i) tmp[i] = a[i] = read();

    sort(tmp + 1, tmp + n + 1);
    int tot = unique(tmp + 1, tmp + n + 1) - (tmp + 1);
    for(int i (1); i <= n; ++i)
        a[i] = lower_bound(tmp + 1, tmp + tot + 1, a[i]) - tmp;
    for(int i (1); i <= n; ++i) ++b[a[i]];
    for(int i (1); i <= n; ++i) sum[i] = sum[i - 1] + b[i];

    for(int i (1); i <= n - 2; ++i) jc1 = 1LL * jc1 * i % mod;
    jc2 = 1LL * jc1 * (n - 1) % mod * n % mod;

    for(int i (1); i <= n; ++i) (cnt += 1LL * b[i] * sum[i - 1]) %= mod;
    LL ans (1LL * cnt * power(jc2, mod - 2) % mod * (n >> 1) % mod * jc1 % mod);
    printf("%lld", ans);

    return 0;
}

它等价于min(a,A.b)<min(b,A.a)min(a, A.b) < min(b, A.a)min(a,A.b)<min(b,A.a)（没有严格证明，但应该对），就hack这个式子！

struct P
{
	int a, b;

	inline bool operator < (const P &A) const
	{
		return max(a + b + A.b, a + A.a + A.b) < max(A.a + A.b + b, A.a + a + b);
	}
} p[N];

hack:
1 1
3 5
2 7

看这个数据之前，先了解c++的sort函数，使用这个函数，重载的比较运算符必须满足严格弱序，即以下4条性质：

  • 非自反性：x不能 < x

  • 非对称性：如果x < y，y不能 < x

  • 传递性：如果x < y && y < z，则x < z

  • 不可比的传递性：（暂用>表示不<）

    由x > y && y > x && y > z && z > y可以推出 x > z && z > x

这个hack数据既是hack了上面min的比较不满度“不可比的传递性”（1 1与3 5“相等”且与2 7“相等”，但3 5 > 2 7，所以sort函数排出来的是不能严格满足min限制的

如何修改？当min相等的时候判断一下，a小的在前即可（a是对答案一定有贡献的），正确性可以用考试技巧2来验证

Code：

struct Node
{
    int a,b;
    bool operator<(Node& y)
    {
        return min(a,y.b)==min(b,y.a)?a<y.a:min(a,y.b)<min(b,y.a);
    }
