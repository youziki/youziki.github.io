牛客IOI周赛18 B-拆分

题意：

牛牛准备送给牛妹一条项链，但是项链太长了，于是他准备将项链拆掉。
牛妹有一些不喜欢的数，所以拆分出来的长度中一定要有至少一个不是牛妹不喜欢的（不然就送不出去了）。
注意，由于项链上的每一颗珠子都不同，所以1 5 1和1 1 5是两种不同的拆分方案，不拆也是一种方案。

$N <= 10^18，P <= 10^9 ，K , T <= 100 $

Solution：

答案=总方案数-不合法方案数

总方案数：考虑每两个数之间都有一个“隔板”，每个隔板可放可不放，方案数为2n−12^{n-1}2n−1

不合法方案数：即“所有数都为给定数”的方案数，不难得出转移方程f[i]=∑j=1kf[i−t[k]]f[i] = \sum_{j=1}^{k} f[i - t[k]]f[i]=∑j=1k​f[i−t[k]]，时间复杂度：O(nk)O(nk)O(nk)

优化：不难发现转移固定，矩阵快速幂即可

另外注意，本题直接AnA^nAn即为答案，不需要初始矩阵

总结：这题里有很多对正解的提示了，

  • “至少有一个不是给定数”，可以想到是正难则反，找不合法的方案数

  • “K,T<=100K,T <= 100K,T<=100，是提示可以用矩阵（值和数量大了都不行了）

总时间复杂度：O((maxT)3logn)O((maxT)^3logn)O((maxT)3logn)

  1. 处理转移矩阵aaa

  4. 矩阵快速幂

P3193 [HNOI2008]GT考试

题意：

有一个不吉利串，求长度为n的不包含这个不吉利串的方案数

Solution：

想朴素暴力，令f[i][j]f[i][j]f[i][j]表示“前i个位置，最后j个和前j个匹配”的方案数，转移：i, j到i+1, k暴力枚举小于等于j+1的k，然后O(n)O(n)O(n)验证，可以发现对于一个后缀，它加一个固定字符可以匹配到的字符是固定的（因为只与这个后缀有关），所以可以将这个转化关系变为一个矩阵。

总体思路长这个样子：先KMP处理出来border，再类似KMP处理出转移矩阵，然后处理出来初始矩阵：f[0][0],f[0][1]...,f[0][m−1]f[0][0],f[0][1]...,f[0][m-1]f[0][0],f[0][1]...,f[0][m−1]，最后矩阵快速幂得出最终状态，将所有f[n][0],f[n][1],...,f[n][m−1]f[n][0],f[n][1],...,f[n][m-1]f[n][0],f[n][1],...,f[n][m−1]加起来即为答案

Code：

#include <bits/stdc++.h>
using namespace std;
#define int long long

const int N = 25;

int n, m, k, nxt[N];
char ch[N];
int a[N][N];

inline void kmp()
{
    for(int i = 2, j = 0; i <= m; ++i)
    {
        while(j > 0 && ch[i] != ch[j + 1]) j = nxt[j];
        if(ch[i] == ch[j + 1]) ++ j;
        nxt[i] = j;
    }
    
    for(int i = 0; i < m; ++i)
        for(char c = '0'; c <= '9'; ++c)
        {
            int j = i;
            while(j > 0 && ch[j + 1] != c) j = nxt[j];
            if(ch[j + 1] == c) ++ j;
            if(j < m) ++ a[i][j];
        }
}

inline void mul(int C[][N], int A[][N], int B[][N])
{
    static int t[N][N];
    memset(t, 0, sizeof t);
    
    for(int i = 0; i < m; ++i)
        for(int j = 0; j < m; ++j)
            for(int l = 0; l < m; ++l)
                (t[i][j] += A[i][l] * B[l][j]) %= k;
    
    memcpy(C, t, sizeof t);
}

inline int power()
{
    int f[N][N] = {1}, ans = 0; // f0
    
    for(; n; n >>= 1, mul(a, a, a))
        if(n & 1) mul(f, f, a);
    
    for(int i = 0; i < m; ++i) (ans += f[0][i]) %= k;
    
    return ans;
}

signed main()
{
    cin >> n >> m >> k;
    cin >> ch + 1;
    
    kmp();
    cout << power();
    
    return 0;
}

总结：加强对固定转移关系的敏锐度、练习一下魔改的KMP也是极好的

P3597 [POI2015]WYC

题意：

给定一张n个点m条边的带权有向图，每条边的边权只可能是1，2，3中的一种。将所有可能的路径按路径长度排序，请输出第k小的路径的长度，注意路径不一定是简单路径，即可以重复走同一个点。

Solution：

考虑朴素暴力：从小到大枚举每个长度的路径的方案数，第一个大于k的长度即为答案

凡一步一步跳的都考虑倍增

于是问题变成了长度2^i次方的路径条数，可以用矩阵快速幂快速求出

只剩下一个问题：这样要求所有边权为1，题中边权很小，考虑加点将一条边权大于1的边变为若干边权为1的边

代码不难
