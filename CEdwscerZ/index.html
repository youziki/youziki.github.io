T150909 老园丁与小司机

先分开每一位考虑，对于区间内有1的，显然可以改，不论xor值中这一位为0还是1，答案加上对应的就可，线段树可维护，时间复杂度：O(nlog2n)O(nlog^2n)O(nlog2n)

那么如果要消去一个log呢？考虑区间内有1即可，求和有些浪费，修改也有些浪费

想象目标：维护一个正常的线段树，需要支持区间xor，查询区间第i为有没有1，查询区间xor值

区间xor值可以O(1)修改，如何O(1)维护区间是否有1？

维护两个变量，记录区间是否有0，是否有1，就可O(1)修改了，因为每次修改是位对齐的操作，所以分别用一个数的不同二进制位即可保存，时间复杂度：O(nlogn)O(nlogn)O(nlogn)

以下完整代码为第一个做法，第二个代码不难写，有个小Trick：

void Xor(int v,int len){
	if(len&1) x^=v;
	int ex0=ao&(~v),exi0=iao&(~v);
	ao&=v,iao&=v;
	swap(ao,iao);
	ao|=ex0,iao|=exi0;
}

这个代码可以手模出所有情况之后，模拟写，这里写法比较取巧，v为1的位，ao与iao是要互换的，其他位不变，ex0记录ao不变的位，exi0记录iao不变的位，ao&=v,iao&=v是将这两个不变的位抹去，也就是只剩下需要变的位，交换，然后将原数不变的位|回去

T150908 求导

理解起来几乎没有难度，就是代码，而且表达式树也是学长说的“应该知道的东西”

这题重点为：

  • 建表达式树，学习代码

    形态概述：

    叶子节点为变量，其他节点为符号，括号不占节点

    总体思路：

    每次都找到最后一个运算的运算符，递归左右子树，最后一个运算的运算符为括号外的最右边的加减号，如果没有，乘除号，依次类推

    每次都传一个完整表达式，x表示当前要处理的左端点，y表示当前要处理的右端点右边一个点，返回的是该表达式的子树的根

    如果当前长度小于1，返回0

    如果当前长度为1，建立一个叶子节点，返回

    从左到右扫，用c1记录最右边的加减号，c2记录最右边的乘除号，用p记录是否在括号内

    如果c1<0，说明没有加减号，c1=c2，如果还为0，说明没有乘除号，说明表达式被括号括起来，减去括号

    现在c1是最后运算的运算符，递归即可

int build_tree(char *s,re int x,re int y){

  • 一些注意点

    在本题中，常数的长度是有可能大于1的，也就是连着几个位置表示一个数，要特判，求值的时候，如果一个点为叶子节点，它可能是变量或常数

    建树时记得考虑乘方和sin/cos运算

    指针从左向右扫的时候，记得写上&& !p，表示在括号外

    有关求导，可以用两个函数返回导函数和函数值

    注意函数嵌套时的导是要特殊计算的，对于+,−,∗,/+,-,* ,/+,−,∗,/，它们都需要左右两边是表达式，所以无法嵌套，只需要考虑乘方，注意：一定不要忘记sin/cos是自带括号的，所以sin/cos也要考虑上
