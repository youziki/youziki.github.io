说一下让我觉得很好的一个细节：

先看我一开始的代码：

m=f[m];

发现过不了样例，对着样例可以发现问题：第二天用不完所有的钱

于是改了之后成为下面这个：

int ans=m;
while(f[ans-1]==f[m])--ans;
m=f[ans]+m-ans;

发现84tps，没有样例了，看代码~~~ 看着这个while循环怎么这么假FAke？

于是想，ans如果继续--，如果减得很多，而f[ans]减得比较少呢？

于是就有了下面的AC代码：

int ans=m;
for(re int j=1;j<=m;++j)ans=max(ans,f[j]+m-j);
m=ans;
