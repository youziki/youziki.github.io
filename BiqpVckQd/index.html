A CF1201D Treasure Hunting

题意：

n×mn \times mn×m矩阵，有一些宝藏，只有在一些特定的列可以向上走，任何时候均可以向左右走，求最少走到所有宝藏步数

Solution：线性dp

显然dpdpdp的对象就是每一列两边端点是从左边上去还是从右边上去（一共4种情况）

实现：肯定是要预处理每一行左右两端位置以及其左右边相邻特定列的位置，dpdpdp直接dpdpdp即可，状态存下4种情况，复杂度O(16n)O(16n)O(16n)。因为转移的时候不一定是从哪一个位置转移上来的，所以对每一个位置都要预处理左右边相邻特定列的位置。代码比较麻烦，第一行和最后一行要特判，并且注意一行没有宝藏的情况。

C CF730J Bottles

题意：

有一些已知容量和水量的水瓶，求装下所有水最少需要的水瓶数，以及此情况下水的最少转移次数（一单位水一次转移即为1）

Solution：背包

这题怎么看，都不止“一维”，这是处理的难点，“要在一维最优的前提下对第二维dp”可以考虑用pair，因为pair自带的比较大小就是先比第一维，所以这样可以保证fff数组的代价是“最小的”。本题中第二维是从0开始，一直到sumsumsum结束，可以保证容量总和大于等于sumsumsum

memset(f, 0x3f, sizeof f);
f[0][0] = {0, sum};
for(int i = 1; i <= n; i++)
	for(int j = 0; j <= sum; j++)
	{
		f[i][j] = f[i - 1][max(j - b[i], 0)];
		f[i][j].first ++, f[i][j].second -= a[i];
		f[i][j] = min(f[i][j], f[i - 1][j]);
	}
cout << f[n][sum].first << ' ' << f[n][sum].second << endl;

J CF633D Fibonacci-ish

题意：

在一个长度为nnn的序列中任选2个数作为斐波那契数列的前两项，求这个序列能选出的最长斐波那契数列长度

Solution：

由于n<=1000n <= 1000n<=1000，直接暴力枚举前两项找最长长度即可，复杂度O(n2l)O(n ^ 2 l)O(n2l)

由于斐波那契数列的数可以当做每次乘上2（除了开头选0 0），所以lll应该是log(1e9)log(1e9)log(1e9)级别的

实现：这题细节很多，很深

  • 首先枚举的时候注意不是枚举有序数对，而是无序数对（对第四项的取值有影响）

  • 要特判开头0 0的情况，后面就要保证不会出现0 0

  • 确定开头两个数后找长度模拟，每次选完在map中减去，最后再都加回来

    这样是为了避免特判很多情况，因为考虑以下情况：第一个数为0，第二个数不为0，第三个数与第二个数相同（数量应不小于2），而如果第一个数不为0，第二个数为0，第三个数和第四个数都与第一个数相同（数量应不小于3），所以就会比较复杂（而且还有未发现的隐含错误）

K CF985E Pencils and Boxes

题意：

一堆数，任意分组，是否有可能满足以下条件：

  • 每个数在一组中

  • 每组至少k个数

  • 每组数最大值最小值之差不超过d

Solution：

是对值域的划分，先排序，然后变为选段满足后两个条件。整体思路考虑dp，令f[i]f[i]f[i]表示到i化为一段是否可行/方案数（这两个是等价的，后来改成了前者），状态需要初始化f[0]=1,f[k]=a[k]−a[1]<=df[0] = 1, f[k] = a[k] - a[1] <= df[0]=1,f[k]=a[k]−a[1]<=d，下面要解决的问题就是转移，基础思路就是找出前一段最后一个点

暴力思路：每个位置从i−ki-ki−k往前一个一个找，碰到f[j]>0f[j] > 0f[j]>0的就停止，会被n=500000k=100000d=1n = 500000 k = 100000 d = 1n=500000k=100000d=1卡

优化：二分最前的合法位置，直接找这个位置到i−ki-ki−k位置的f[]f[]f[]前缀和

实现：

  • 注意二分完之后，可以根据lll的取值，轻松特判出不合法和全选的情况

  • 如果直接将f[i]f[i]f[i]赋为前缀和之差，sumf[]sumf[]sumf[]会爆longlonglong longlonglong（CF不让用__int128），所以要赋为1/0

int n, k, d, a[N], f[N], sumf[N];

signed main()
{
    n = read(), k = read(), d = read();
    for(int i = 1; i <= n; ++i) a[i] = read();
    
    sort(a + 1, a + n + 1);
    
    f[k] = a[k] - a[1] <= d, f[0] = 1;
    for(int i = 0; i <= k; ++i) sumf[i] = 1;
    sumf[k] += f[k];

    for(int i = k + 1; i <= n; ++i)
    {
        int l = 0, r = i - k + 1;
        while(l < r)
        {
            int mid = (l + r) >> 1;
            if(a[i] - a[mid + 1] <= d) r = mid;
            else l = mid + 1;
        }
        
        if(l == i - k + 1) f[i] = 0;
        else if(l == 0) f[i] = 1;
        else f[i] = (sumf[i - k] - sumf[l - 1]) > 0;
        sumf[i] = sumf[i - 1] + f[i];
    }
    
    puts(f[n] > 0 ? "YES" : "NO");
    
    return 0;
}

M CF766D Mahmoud and a Dictionary

题意：

给一些单词，它们可能是同义或者反义，给出一些关系定义，从前面的定义开始建立关系，如果有的关系定义和之前的冲突输出NO,否则输出YES。然后查询q次单词x和单词y的关系（同义/反义/无关）。

Solution：

扩展域并查集板子，如果A，B矛盾，就A与B+n连一条边

for (int i = 1; i <= n; i++)

P2642 双子序列最大和

题解有几篇就行了，我写一下自己的总结

是一道黄题，但我并没有秒掉，还想了十来分钟还是个hongming，最后想出来也是没什么思考过程/kk

思维被传纸条带偏了，这么大的范围，老想着用状态将选的方案表示出来。但我们看到，正解的思维是线性的（包括状态和转移），对吧，我还是思维太僵化了

像这题有双子串，如果状态比较复杂，可以考虑dp部分，结合其他技巧，状态少了，就好表示了

如果只有一个连续子串呢？没必要完全表示子串，是吧，考虑到转移肯定是从前面一个地方转移的，问题就迎刃而解了

因为忘了最大连续子串了，所以这题相当于0基础想的思路，也是正好发挥了这道题的价值吧

总结：

状态复杂，可以考虑用dp结合别的技巧（像“中间还要间隔一个”之类的，用别的技巧会很简单）

状态设计可以先考虑考虑转移（虽然听起来有点鬼），没必要把状态都表示出来

找到dp的子问题

不要忘记线性dp的状态呀（最后一个位置以前...）
