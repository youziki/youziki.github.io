为什么这么好的题洛谷没有

题意：（建议看原题面，因为这题考一点阅读理解）

求只有互不相交的简单环的n个点的无向连通图个数，膜

Solution：

先将题分为两部分：

  1. 找出m个块（环）

  2. 求n个点m个块的图的生成树个数

先来解决第二个部分：

显然对于一个图，我们通过purfer编码求它的生成树个数，先随便定一个根节点，然后它的purfer序列有m-2个位置，对于每一个位置，有n种可能，为什么不是m种呢？看图：

如果当前最小叶子是2（当然现在还不是），它可能会是4的儿子，5的儿子或者6的儿子，所以下一个位置的选择中一个环中的点(4,5,6)是独立的

于是，第二个问题解决了！

再来看第一个部分：

考虑dp解决，设f[i][j]表示i个（没有说“前i个”是为了区分序列的有序和i个点的无序），分为j块方案数

考虑第一个点所在的块有k个点，方案数 ×f[i−k][j−1]\times f[i-k][j-1]×f[i−k][j−1]

注意：k的范围：1~i-j+1（因为后面的j-1个块至少要j-1个点）

这个方案有多少个？C(i,k)C(i,k)C(i,k)？错了！

因为是第一个点所在的块，所以显然要有第一个点，所以应该为C(i−1,k−1)C(i-1,k-1)C(i−1,k−1)。然鹅还是错的。

因为有了k个点之后，组成的环不确定！显然方案数还要乘上一个k的圆排列

k的圆排列：(k−1)!2\frac{(k-1)!}{2}2(k−1)!​

证明：考虑随便排列k，因为在图中，旋转、翻折无意义，所以要除以k再除以2

这就可以直接做了！然鹅，还是错的。。

考虑这道题中，我们一个环它不是孤立的，它是要往外连边，对于一个确定了的排列生成一个环，选择哪一个点往外连边是不同的，也就是方案数还要乘上一个k。或者说也可以将每次旋转理解成是换一个向外连边的点，而翻折还是无意义，所以不需要除以k，只需要k!2\frac{k!}{2}2k!​

然鹅，这样不会重复吗？我的意思是，我们在用purfer序列求方案数的时候用到了“不同点作为父亲”时的区别，这里又区分一个环内的点，不会重复吗？

是不会的，对吧。。因为确定了一个根节点之后，整张图就成了一个有向图，所以对于一条边，purfer那个用的是它的终点处讨论，但终点相同起点不同也是不同的情况，所以在这里，讨论“往外连边的点”实际上就是在考虑不同的起点

那么就有一个疑问：一个块是有很多条连出的边的，那岂不是要讨论每条边在该块内的起点？（这个问题困扰了我一段时间）

实际上也是不用的。因为对于一条边，我们把它看成是从叶子指向父亲的，这样一个块就只有一条出边了，而每条边终点处的分类讨论（也就是一个点儿子们指向它的边）我们在用purfer统计答案时就计算过了，所以以上方法已经可以将每条边的所有情况计算进去，不重不漏

（对于以上5段以及“用purfer序列计算部分2时n为底数的解释”确实比较抽象，不易表达成具体的话，请读者性感理解）

总结一下，第一部分dp的递推关系：

∑i=1i−j+1f[i−k][j−1]×C(i−1,k−1)×k!2\sum_{i=1}^{i-j+1}f[i-k][j-1] \times C(i-1,k-1) \times \frac{k!}{2}∑i=1i−j+1​f[i−k][j−1]×C(i−1,k−1)×2k!​

O(n2)O(n^2)O(n2)的状态，O(n)O(n)O(n)的转移，总时间复杂度：O(n3)O(n^3)O(n3)

于是写代码就好了！

实现：

你以为这样就能切？然鹅，，还是错了。。。

考虑如果只有一个块，在求圆排列的时候就不需要乘上一个k了，因为不能向外连边了！所以代码中ans一开始是等于g[n−1]g[n-1]g[n−1]而不是g[n]g[n]g[n]，相当于ans=g[n]nans=\frac{g[n]}{n}ans=ng[n]​了

我们根据经验考虑到肯定还有坑。。。。是的

考虑在本题中2是没有圆排列的！因为按照题里的定义，如果只有两个点想构成一个环的话，需要a1,a2之间有两条边（既是中间部分要求，又有首尾要求），而题中保证了没有重边和自环，所以这种情况不存在，预处理圆排列时赋为0

话说，我现在在1点多的时间，大骂出题人毒瘤！！！这题实现居然还有坑？！害得我调不出来 （于是来加上这一段）

那就是如果预处理阶乘和逆元求组合数是错的，因为模数不是质数！所以不一定有逆元！要O(n^2)预处理组合数

Code：

#include<bits/stdc++.h>
#define int long long
using namespace std;
int n,mod,g[210],C[210][210],f[210][210];
void init(){
	for(int i=0;i<=n;++i)
		for(int j=0;j<=i;++j)
			if(!j)C[i][j]=1;
			else C[i][j]=(C[i-1][j]+C[i-1][j-1])%mod;
	g[1]=1,g[2]=0,g[3]=3;
	for(int i=4;i<=n;++i)g[i]=g[i-1]*i%mod;
}
void dp(){
	f[0][0]=1;
	for(int i=1;i<=n;++i)
		for(int j=1;j<=i;++j)
			for(int k=1;k<=i-j+1;++k)
				f[i][j]=(f[i][j]+f[i-k][j-1]*C[i-1][k-1]%mod*g[k])%mod;
}
signed main(){
	cin>>n>>mod;
	init();
	dp();
	int ans=g[n-1];
	for(int i=2,p=1;i<=n;++i,p=p*n%mod)ans=(ans+f[n][i]*p)%mod;
	cout<<ans;
	return 0;
}
