题解 P2816 宋荣子搭积木

2020-10-30

严格O(n)O(n)O(n)

废话：为了卡常，我想出来O(n)O(n)O(n)做法！

铺垫：

不解释O(nlogn)O(nlogn)O(nlogn)，应该也是所有人都能想到的做法！（为什么会有人用O(n2)O(n^2)O(n2))

Code 1.0：

int n,x[5005];
bool check(int mid){
	for(int i=1;i<=n;++i){
		int j=i;
		while(x[j+1]==x[i])++j;
		if(j>(x[i]+1)*mid)return 0;
		i=j;
	}
	return 1;
}
int main(){
	scanf("%d",&n);
	for(int i=1;i<=n;++i) scanf("%d",x+i);
	sort(x+1,x+n+1);
	int l=0,r=n;
	while(l<r){
		int mid=(l+r)>>1;
		if(check(mid))r=mid;
		else l=mid+1;
	}
	printf("%d\n",l);
}

显然，sort太慢了，题中给了x[i]<=nx[i]<=nx[i]<=n，用桶排代替，那么又能想到check函数中的forforfor循环里可以简化

Code 2.0：

int n,b[5001];
bool check(register int mid){
	register int sum=0,i=0;
	while(i<=n){
		sum+=b[i];
		if(sum>(++i)*mid)return 0;
	}
	return 1;
}
int main(){
	n=read();
	for(register int i=1;i<=n;++i) ++b[read()];
	int l=0,r=n;
	while(l<r){
		int mid=(l+r)>>1;
		if(check(mid))r=mid;
		else l=mid+1;
	}
	printf("%d\n",l);
	return 0;
}

正题：

仔细看while循环，想象自己是个抠门的人，每次check求sum是不是太浪费了，O(n)O(n)O(n)预处理前缀和又不舍得。。。

发现可以换一种表述：“对于每一个位置i，最终答案×i+1\times i+1×i+1应该大于等于i位置的前缀和”

那么这就很像一个O(n)O(n)O(n)的问题了，我们再来想一个简洁的实现

Code 3.0：

inline int ma(register int a,register int b){return a>b?a:b;}
int b[5001];
int main(){
	int n(read()),ans(0);
	for(register int i(1);i<=n;++i)++b[read()];
	for(register int i(0),sum(b[0]);i<=n;sum+=b[++i])ans=b[i]>ans?ma((sum+i)/(i+1),ans):ans;
	printf("%d",ans);
}
